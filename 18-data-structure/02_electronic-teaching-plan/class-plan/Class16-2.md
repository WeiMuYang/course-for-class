## 4 内部排序    

### 4.1 选择排序   
>[简单选择](../../05_data-structure-code/9-排序/06-简单选择排序.cpp)   
1. 首先通过 n –1 次关键字比较，从 n 个记录中找出关键字最小的记录，将它与第一个记录交换。     
2. 再通过 n –2 次比较，从剩余的 n –1 个记录中找出关键字次小的记录，将它与第二个记录交换。    
3. 重复上述操作，共进行 n –1 趟排序后，排序结束。     


>[堆排序](../../05_data-structure-code/9-排序/07-堆排序.cpp)   
1. 建堆    
2. 筛选    
但是我们是先讲筛选再讲建堆，因为筛选的过程类似于建堆过程，将小根堆建成了大根堆     
3. 对深度为 k 的堆，“筛选”所需进行的关键字比较的次数至多为 2(k-1)；在每一层上先和兄弟比，再和双亲比，直到根结点为止，只在根结点那一层没有比较    
4. 对 n 个关键字，建成深度为$ h(=\lfloor log_2n\rfloor+1) $的堆，所需进行的关键字比较的次数至多 4n；   
5. 调整“堆顶” n-1 次，总共进行的关键字比较的次数不超过   
  $$  2 (\lfloor log_2(n-1) \rfloor+\lfloor log_2(n-2)\rfloor+ …+\lfloor log_22\rfloor) < 2n(\lfloor log2n\rfloor) $$  
  调整时调用建堆函数n-1次，分别是去掉第n-1个结点，直到去掉第2个结点，每次比较$ 2(\lfloor log_2(n-1)\rfloor +1 -1 ) $次，总体求和即可。

### 4.3 基数排序   
**最低位优先法**  
>链式基数排序，具体作法为：    
1. 以静态链表存储待排记录，并令表头指针指向第一个记录；   
2. “分配” 时，按当前“关键字位”所取值，将记录分配到不同的 “链队列” 中，每个队列中记录的 “关键字位” 相同；     
3. “收集”时，按当前关键字位取值从小到大将各队列首尾相链成一个链表；    
4. 对每个关键字位均重复 2 和 3 两步。     

>效率分析：   
n —— 记录数  
d —— 关键字位数  
rd —— 关键字取值范围 （如十进制为10）    
分配（每趟）：`T(n)=O(n)`   
收集（每趟）：`T(n)=O(rd)`     
总的时间复杂度：`T(n)=O(d(n+rd))`(一次分配和收集只能解决一个关键字)      
辅助空间：  `f(n)=2rd 个队列指针 + n 个指针域空间`   
空间复杂度：`S(n) = O(rd)`(因为收集的过程就是通过循环多次调用实现的，因此队列指针是空间复杂度)
